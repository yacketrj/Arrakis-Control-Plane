```text
Summary:
First controlled pre-1.0 release candidate for DA Manager.

Scope:
- secure localhost-default admin foundation
- Discord auth/session/link groundwork
- read-only Player 360 foundation
- inventory request/order coordination
- Farming Requests UI
- Inventory Studio action-history and stack-size groundwork
- AppSec endpoint audit foundation
- generated route auth-boundary coverage
- high-risk/destructive mutation audit-event coverage
- blocked-mutation audit coverage
- route-specific audit target metadata coverage
- changelog/ledger compaction

Out of scope:
- Discord-driven full server management
- Live Admin RMQ command execution
- Welcome Kits
- raw command publishing UI
- Player 360 mutation/self-service expansion
- formal SOC 2 or ISO certification readiness
```

## Rollback plan

```text
Rollback trigger:
- release candidate validation fails in a way that cannot be fixed safely before tagging
- new critical/high security issue is identified
- update script cannot produce repeatable build artifacts

Rollback steps:
1. Do not create or publish the tag.
2. Revert release setup commits if necessary.
3. Return VERSION to the prior value or dev state if release is abandoned.
4. Record the failed candidate in PATCH_NOTES.md and the release checklist.

Expected rollback duration:
15–30 minutes for docs/config-only rollback.

Data compatibility concerns:
None for release setup. No schema or player-data mutation was added.

Operator communications:
Notify operators that v0.1.0-rc.1 is deferred and continue using the previous validated commit.
```
